How to compile DARWIN with MSVC++ (This is only tested in MSVC++ 6.0)

  1. First, get gtk, glib, gdk, and extra libs for windows. (There's a link to this stuff on www.gtk.org)
  2. Unzip those files, and put all the .dll files in c:\windows\system.

    Put all the .h files in C:\Program Files\Microsoft Visual Studio\VC98\Include
    (gtk .h files need to be in \gtk subdirectory, and gdk .h files in \gdk subdirectory)

    Put all the .lib files in C:\Program Files\Microsoft Visual Studio\VC98\Lib
  3. If you want to test it out, try to compile a hello world type program at this point.  (Add gtk-1.3.lib, gdk-1.3.lib, and glib-1.3.lib [or whatever the names of the .libs you got are] to the libraries section in the link tab of the Project->Settings... dialog.)
  4. Load the workspace in the \msvc DARWIN subdirectory.  Edit config.h and comment out the ENABLE_NLS and HAVE_SYS_UTSNAME_H lines if they exist.  If you want, edit the BUILD_ALIAS to reflect your computer (e.g #define BUILD_ALIAS "i586-microsoft-windows98").
  5. You may need to add the source directories to the Directories tab in the Tools->Options dialog.
  6. Try building the project.  You'll almost certainly get a number of warnings about symbols being truncated in debug info.  You may also encounter any number of problems here, such as MSVC++'s broken variable scoping rules, etc.  You're on your own here, though.
